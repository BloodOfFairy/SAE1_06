Définition du logiciel libre :

Pour commencer, le logiciel libre désigne des logiciels qui respectent la liberté de l’utilisateur. En effet, cela permet aux utilisateurs la liberté de copier, distribuer, modifier, d’améliorer les logiciels sans contrepartie financière. Il y a des droits pour ces libertés qui sont établies sous forme de licence « libre ». Ces licences sont basées sur les droits d’auteur.

La définition du logiciel libre est basée sur 4 libertés essentielles en tant qu’utilisateur :

La liberté d’exécuter un programme comme l’on désire, par tous les usages.

La liberté d’étudier le fonctionnement du programme et de le modifier pour effectuer des tâches informatiques. L’accès au code source est donc nécessaire.

La liberté de redistribution des copies, d’aider les autres. Cela permet de donner ou de vendre des copies.

La liberté d’améliorer un programme et de publier ses changements. L’accès au code source est nécessaire.

Définis dans les années 1980 par Richard Stallman la FSF ou Free Software Foundation précise quelques points :

  • Comme le fait que ces libertés doivent être irrévocables

  • Possibilité d'en jouir sans devoir prévenir quelqu’un

    • Possibilité de redistribuer le programme sous toute forme, notamment compilée, et le code source doit être accessible pour jouir des libertés d'étude et d'amélioration

    Possibilité de fusionner des logiciels libres dont on n'est pas soimême l'auteur.

    La FSF accepte des restrictions mineures dont un logiciel modifié doit être représenté lorsqu'il est redistribué.

Histoire logicielle libre :

L’histoire du logiciel libre, Richard Stallman reflète un grand problème actuel dans les entreprises qui ferment l’accès au logiciel. Lors d’un bug d'imprimante (bourrage papier), Richard tente de réparer ce problème. Pour cela, il demande un accès au code source de l’imprimante, mais celuici est refusé. Face à cet échec, il décida de créer un système d'exploitation complet et libre construit uniquement sur le partage des données appelée GNU/Linux.

D’où sa citation : "Les éditeurs de logiciels cherchent à diviser et à conquérir les utilisateurs, en interdisant à chacun de partager avec les autres. Je refuse de rompre la solidarité avec les autres utilisateurs de cette manière.”

C’est pour cela que dans les années 1980, Richard Stallman formalise la notion de logiciel libre. Dans la fin des années 1990, les logiciels libres connaissance un succès comme GNU et LINUX. Les entreprises d’informatiques et les média suscitent un grand intérêt pour ces derniers.

Logiciel libre et open Source :

Il est important de savoir faire la différence entre un logiciel libre et un l’Open Source.

L’appellation « Open Source » fait référence au logiciel Open Source. Un logiciel Open Source est un logiciel conçu pour être accessible à tous et représenter sous forme de licence. Il permettra à toutes les personnes de le consulter, le modifier et copié le code pour en faire ce que l’on veut. Ce type de logiciel peut être construit en collaboratif avec d’autres personnes. Ce type de logiciel n’est pas très cher, est plus maniable et plus constant que les autres. Ils sont développés par la communauté et non par les entreprises. En 1998 unes des organisations, « L’Open Source Initiative » fut fondée sous l’impulsion de Bruce Perens et Eric Raymond. Cette organisation est dévouée à faire promouvoir les logiciels open source. Elle est compatible avec la coexistence de logiciels dont le code est ouvert, mais n'est pas libre. C’est pour cela qu’un logiciel open source est un logiciel dont le code source est en accès.

Pour pouvoir dire qu’un logiciel est un logiciel open source, il doit respecter les 10 principes du mouvement open source.

1) Redistribution libre

La licence n'empêche aucune partie de vendre ou de céder le logiciel.

2) Code source

Le programme doit inclure le code source et doit permettre la distribution tant du code source que de sa forme compilée.

3) Travaux dérivés

La licence doit permettre les modifications et les travaux dérivés, ainsi que leur distribution dans les mêmes conditions que la licence du logiciel original.

4) Intégrité du code source de l'auteur

La licence doit autoriser la distribution de logiciels construits à partir de code source modifié. La licence ne peut pas accepter les travaux dérivés.

5) Pas de discrimination contre les personnes ou les groupes

La licence ne doit pas discriminer une personne ou un groupe de personnes.

6) Absence de discrimination dans les domaines de compétence

La licence ne doit pas empêcher une personne ou un groupe à utiliser le programme dans un domaine d’activité spécifique.

7) Distribution de la licence

Les droits attachés au programme doivent être les mêmes et être appliqué à tous les programmes.

8) La licence ne doit pas être spécifique à un produit.

Les droits attachés au programme ne doivent pas dépendre de son appartenance à une distribution de logiciel particulière.

9) La licence ne doit pas restreindre d'autres logiciels.

La licence ne doit pas imposer de restrictions à d'autres logiciels distribués avec le logiciel sous une licence.

10) La licence doit être neutre sur le plan technologique.

La licence ne peut pas être basée sur une technologie ou un style d'interface individuel.

La différence entre un logiciel libre et un logiciel open source n’est pas facile à définir. À la fin des années 1990, la communauté des logiciels libres ont commencé à dire « logiciel open source » au lieu de « logiciel libre ». Cela crée deux mouvements : le mouvement du logiciel libre qui est social et le mouvement de l’open source qui permet de développer et de diffuser les logiciels. Le mouvement du logiciel libre soutient les valeurs philosophiques et politiques. Selon Richard Stallman, « un logiciel libre est un mouvement social et l’open source est une méthodologie de développement ».

Un logiciel libre est forcément open source, mais un logiciel open source ne peut être un logiciel libre. À l’inverse, ce sera un logiciel privatif (qui ne permet pas de copier, redistribuer, de modifier comme l’on désire) que l’on va avoir.
